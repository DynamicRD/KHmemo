엔티티->테이블
이름->칼럼명
논리적 모델링

데이터모델링 : 현실세계를 데이터로 컴퓨터화 시킨다

논리적 모델링: 모델하우스를 짓는다 (가짜집)

개념적 모델링: 어디다 집을 어떻게 지을지 상세히 설명

물리적 모델링: 진짜 지었다

논리적 모델링에서 관계 프라이머리키 나온다
개체(엔티티,테이블명) 관계(1:1,1:n,n:m) 속성(밑줄이 들어가면 프라이머리키) 

개체(엔티티): 논리적 모델에서의 테이블
속성 어트리뷰트: 논리적모델에서의 칼럼명

다 만들고 로우에 들어가는게 인스턴스

기본키: ERD에선 밑줄로 표시

관계 1:1, 1:N, N:M
카디날리티 비율: 개체를 이루는 각 값이 관계에 참여할 수 있는 수를 나타낸다

Mapping 규칙
ENTITY 		=>	TABLE
ATTRIBUTE	==>	COLUMN
RELATIONSHIP	=>	FOREIGN KEY

1:1관계는 하나의 테이블로 통합한다(주 키가 2개다)
1:n은 1이 메인 N이 서브

right outer join 은 왼쪽에다 null을 채운다,우측 대상 없는것도 보여주준다
