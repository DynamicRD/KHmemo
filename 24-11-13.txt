트랜잭션(try catch와 비슷함)
트랜잭셔니 데이터 처리에 논리적으로 하나의 작업단위
commit,rollback,savepoint

ddl과 dcl은 롤백이 적용이 안된다
dml은 롤백이 된다

트랜잭션은 데이터의 일관성을 유지하고 데이터 복구를 위해

commit하는순간 트랜잭션의 시작과 끝이다

커밋과 롤백: 데이터 무결성, 영구적인 변경전에 데이터 변경사항 확인가능,
		논리적으로 연관된 작업 그룹화 가능

savepoint label_name;
rollback to laber_name;

뷰
물리적인 테이블을 근거한 논리적인 가상 테이블
기존 테이블에 대한 하나의 쿼리문
실제 테이블에 저장된 데이터를 뷰를 통해서 볼 수 있도록 한다
뷰는 데이터 저장공간이 없고 단지 쿼리문을 저장하고있는 객체이다

뷰의 동작원리
사용자가 뷰에서 질의를 하면 user_views에서 뷰에 대한 정의를 조회
기존테이블에 대한 뷰의 접근권한을 살핀다

단순뷰			복합뷰
하나의 테이블로 생성	여러개의 테이블로 생성
그룹함수의 사용불가능	그룹함수의 사용이가능
distinct 사용이불가능 	distinct 사용이 가능
dml사용가능		dml사용불가능
rownum 자동으로 1증가, order by하면 순서 재배치가 일어남
rownum은 조건에 안맞으면 바로 멈춘다
