JSP(보기는 html이지만 사실 서블릿 자바 클래스이다)
	서블릿은 화면표시가 힘들고 분업화가 어려움
로직을 그리는데 최선을 다해야한다
1.톰캣서버 2.웹서버 3. 서블릿컨테이너
톰캣서버: 개인컴퓨터에 서버를 열음(자바로만들어짐)
->깔면 웹서버와 서블릿컨테이너(서블릿클래스)를 만들어준다
웹서버: 외부로부터 요청을받아서 html이면 찾으면 바로 전달함(나머지는 서블릿한테 갖다줌)
	{(웹->서버)스타트라인/(서버->웹)상태라인},메시지헤더,메시지바디로
	404,405,200,300,500
서블릿컨테이너: 서블릿클래스를 객체(생성자,콜백함수)로 만들어 작동, 요청한정보를 파악한뒤 
		웹서버에 전달함

1. 톰캣서버 설치하고 운영을 하면 웹서버와 서블릿컨테이너 생성
2. 클라이언트가 url에 html요청을 하면 웹서버는 받아서 상태정보,메시지헤더(서버정보),메시지바디(html 코드,css,자바스크립트)를
	 클라이언트 에 보내준다 상태정보는 200 300(잘작동) 403 404(요청했는데 없음) 405(두겟이 오버라이딩x) 500(서버에서 예외처리 걸림)번 가지고있음
3. 클라이언트가 jsp나 서블릿을 요청을 하면 웹서버는 서블릿 컨테이너에게 요청
	서블릿 컨테이너는 해당된 서블릿클래스를 메모리를 요청한 순간 로딩함(레이지 로딩)
	서블릿컨테이너가 뉴 해서 객체가 만들어지며 디폴트생성자,(오버라이딩) 아이나이트함수, 디스트로이함수, 서비스함수, 
	두겟, 두포스트 함수
	아이나이트(생성자 만들어지면 1회실행) 디스트로이(제거할떄만) 서비스 두겟 두포스트(항상 작동)
	서비스가 작동할떄마다 스레드가 하나 생성되어서 실행됨
4. 서비스함수 : 컨트롤러 3개. 사용자 요청을 데이터로 받는다. 요청을 보고  db를 접속하고 curd처리. 받은결과값을 화면에 출력
	서블릿이 하는 일. 그리고 만든 화면을 웹서버로 보냄
	겟, 포스트 안에 리퀘스트,리스폰스(브라우저에 전달)가 있다
	1.데이터를 받고 2.db연동해서 curd, 3.결과값을 html에 심어서 웹서버로 보냄 웹서버는 상태 3단계로 만들어서 보냄

jsp와 서블릿의 차이
url 맵핑 = 클래스가 서블릿인걸 알려줌
웹서버가 서블릿컨테이너한테 요청할때 web.xml을 이용해서 객체를 만든다
xml은 하나의 지침서이다
welcome-file-list:	위에서부터 있는 순서대로 실행
servlet-mapping된걸 찾아서 해당 servlet을 레이지 로딩한다음에 객체를 생성
혹은 서블릿에서 어노테이션으로 @WebServlet(name = "myFirstServlet", urlPatterns = { "/myFirstServlet" })

서비스의 리퀘스트는 상태정보,메시지헤더,메시지바디 포함이다 (스타트라인에 겟,포스트방식인지 보여준다)
리스폰스는 아우터스트림,라이터,재요청(리디렉트)이냐
콜백함수 : 오버라이딩하면 자동으로 실행하는 함수
web.xml을 수정하면 톰캣서버를 껐다켜야함

url : http://localhost:8080/jspStudy/welcome.jsp
uri : /jspStudy/welcome.jsp
